Content Telemetry Profiles

Profiles layer community-specific requirements on the neutral Content Telemetry wire format.

The base standard defines event types, schemas, conformance levels, transport guidance, and manifests. Profiles add requirements for a specific community or use case without changing the wire format.

Profile URIs

  • https://contenttelemetry.org/profiles/spur — SPUR publisher-facing profile.
  • https://contenttelemetry.org/profiles/openattribution/commerce — OpenAttribution Commerce profile for agentic commerce.

How to use these URIs

A licensing declaration can reference one of these profile URIs to identify the profile whose requirements apply. The declaration carries licence-scoped demands such as minimum conformance level, privacy floor, cadence, and any profile-specific capability requirements. The telemetry manifest remains the transport and identity declaration.
